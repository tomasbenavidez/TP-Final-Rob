Contratos que no deben mezclarse

Nueve invariantes de integración

Ejecuciones separadasParte A, adaptación y Parte B no se ejecutan simultáneamente.
TF global exclusivoGraph SLAM y MCL son productores alternativos de map → odom.
Dos contratos /landmarksParte A publica MarkerArray; Parte B publica PoseArray.
Frames por plataformaParte A usa base_link; Parte B usa base_footprint.
Segunda pasada realLa corrección SLAM se compone con odometría densa; no se interpola simplemente entre keyframes.
Autoridad de movimientostate_machine es el único productor de /cmd_vel dentro de Parte B.
Sensor sin realimentaciónlandmark_sensor usa odom como verdad simulada, no la estimación de MCL.
Persistencia entre etapasEl mapa estático es el único artefacto persistente compartido: originalmente map.pgm/map.yaml; en la adaptación simulada, map_sim.pgm/map_sim.yaml.
Adaptación, no SLAMEl remapeo del entorno simulado reutiliza la grilla de ocupación: recibe /scan y obtiene su pose con TF odom ← scan_frame; sim_mapper no se suscribe a /odom ni ejecuta Graph SLAM.
